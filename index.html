Geometries

Three.js provides numerous built-in primitive shapes. These mathematically generated shapes form the foundation of any scene.

Interact: Drag the box on the right to orbit the camera around the objects!

Materials

Materials define how surfaces react to light. We have Basic (unlit), Lambert (matte), Standard (PBR), and Physical (advanced PBR with clearcoats).

Notice how the exact same geometry looks drastically different depending on its material properties.

Lighting

Without light, standard materials are invisible. Here we compare an AmbientLight (flat overall lighting), PointLight (like a lightbulb), and a SpotLight.

Shadows

Shadows ground objects in reality. They require enabling shadow maps on the renderer, light (cast), object (cast), and floor plane (receive).

Group Hierarchy

You can group objects together. If you rotate the parent group, all children rotate with it, relative to the parent's pivot. This is perfect for planetary systems, robot arms, or cars.

Watch how the moon orbits the earth, while the earth orbits the sun.

Cameras

Two main camera types exist: Perspective (mimics human eye, objects get smaller in the distance) and Orthographic (flat isometric projection, no vanishing point).

Color Textures

Instead of solid math-based colors, we can map 2D images onto 3D surfaces. This is called a Diffuse Map or Color Map.

PBR & Normal Maps

To fake high-detail geometry without mathematical cost, we use a Normal Map to change how light hits the surface, making flats look bumpy.

Combined with Roughness maps, the surface looks incredibly realistic.

Environment Maps

Shiny objects reflect the world around them. Using a CubeTexture mapping 6 images (like a skybox) provides cheap, realistic reflections.

GLTF Loading

Complex assets are created in tools like Blender and exported as GLTF/GLB files, which bundle geometries, cameras, lights, textures, and animations.

Raycasting

Transforming 2D mouse coordinates into a 3D ray collision line allows us to find exactly what object the user is clicking or hovering over.

Hover over the spheres below!

InstancedMesh

Drawing one box 10,000 times normally freezes the browser because it talks to the GPU 10,000 times per frame. InstancedMesh groups them all into exactly 1 draw call.

Below are 10,000 independently placed cubes.

Particles (Points)

For rain, smoke, and galaxies, rendering thousands of flat textured sprites is vastly cheaper than 3D geometry. We do this using THREE.Points.

Custom Shaders

When you need total control, you bypass Three's materials and write raw GLSL code to execute directly on the GPU for every single pixel.

Here we displace geometry vertices using mathematical sine waves.

Render Targets

You can render a scene onto a secondary texture instead of the screen, and then map that texture onto a mesh object. Perfect for security cameras, mirrors, or TV screens.

Post-Processing (Bloom)

Before showing the final frame to the user, we can pass it through a series of full-screen image filters. UnrealBloomPass makes bright pixels bleed and glow.

Raymarching & SDFs

Instead of polygons, we can mathematically define shapes using Signed Distance Fields and evaluate them via a Raymarching fragment shader. Notice how these spheres melt together like liquid metal without any geometry!

Curl Noise GPU Particles

Moving 100,000 particles on the CPU is slow. By passing time into a procedural Curl Noise mathematical function directly in the Vertex Shader, the GPU can effortlessly simulate mesmerizing, fluid-like particle galaxies.

Volumetric Raycasting

By stepping a ray through 3D space and accumulating noise density within a mathematical volume, we can render incredibly realistic structures like swirling nebulas entirely procedurally in a single shader.

Next-Gen Post-Processing

AAA games combine intense post-processing pipelines to achieve realism. This scene utilizes a complex EffectComposer chain featuring Screen Space Ambient Occlusion (SSAO), Depth of Field (Bokeh), and HDR Bloom.

Rigid Bodies & Colliders

Physics engines handle gravity and collisions. Dynamic bodies yield to forces (like falling cubes), while Fixed bodies remain static (like the floor).

Kinematic Bodies

Sometimes you need an object to move programmatically but still violently push physics objects out of its way. A KinematicPositionBased moving arm demonstrates this.

Joints & Constraints

You can constrain bodies together using mathematical joints. A SphericalJoint creates a pendulum or chain link allowing rotation but securely locking the anchor point. Watch the chain slowly swing into the tower!
